# Adverse impact snapshot

_This file is generated by `npm run adverse-impact:report` (SAMPLE data for the
pitch) or, in production, populated from the auth-gated `GET /admin/adverse-impact`
endpoint over real review decisions from the last 30 days._

```bash
npm run adverse-impact:report   # writes a SAMPLE table here, offline
```

## What it reports

For each neutral cohort dimension (`role_family`, `seniority`, `interview_language`,
`cv_school_class`, `city_class`): applicants, selected (advanced), selection rate,
and the ratio vs the reference cohort (the one with the highest selection rate).

Thresholds (EEOC 1978 four-fifths rule):

- ratio < **0.80** → `fail`
- 0.80 ≤ ratio < **0.90** → `warn`
- ratio ≥ 0.90 → `pass`
- cohorts with fewer than **5** applicants → `warn` (signal too noisy), never `fail`.

> Adverse impact on neutral proxies is an early-warning system, not a substitute
> for a protected-attribute analysis (which requires explicit consent and a DPA).
> AssumerAI does not collect direct protected attributes by design.
